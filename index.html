01 — Introduction

About Me

I write documentation that helps engineers ship faster — API references, SDK guides, onboarding tutorials, and the internal docs nobody else wants to own.

Over the past eight years I've worked across developer tools, fintech, and infrastructure, translating complex systems into prose that respects the reader's time. My approach starts with the task a person is trying to complete, not the feature a team happens to have built.

I'm comfortable reading source code, running the product myself, and interviewing the engineers who built it. Docs are a product surface, and I treat them with the same rigor: tested examples, versioned content, and feedback loops that catch drift before users do.
